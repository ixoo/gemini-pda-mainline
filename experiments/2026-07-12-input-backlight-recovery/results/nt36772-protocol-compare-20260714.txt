Equivalent protocol fields:
- The vendor has one physical I2C client at 0x62. Its CTP_I2C_READ helper
  sends one command byte and reads len-1 data bytes; CTP_I2C_WRITE sends all
  bytes. The backend keeps one client and writes the supplied target directly
  into each transfer message, preserving the vendor's alternate 0x01 target.
- Each vendor trim attempt writes 0x69/35 ms, 0xa5/15 ms, and 0x35/10 ms at
  0x62, selects xdata 0x01f600 at 0x01, then reads command 0x4e. The backend
  performs the same bounded five-attempt sequence and accepts only the live
  NT36772 suffix 72 66 03.
- The vendor probe performs a second 0x69/35 ms bootloader reset after trim
  identification. The backend now does the same before reset-state polling;
  this was a deliberate correction to the first draft.
- Vendor reset-state command 0x60 reads five data bytes after the command and
  accepts state >= 0xa0 and < 0xff. The backend reads the same five data bytes
  with a bounded 100-iteration poll.
- Vendor firmware-info command 0x78 requests 17 bytes including the command,
  leaving 16 data bytes. The backend requests those 16 data bytes, checks the
  version/complement pair, and uses the same data offsets for raw X/Y maxima.
  It fails closed on invalid dimensions instead of applying the vendor's
  1080x1920 fallback.
- The NT36772 map uses event xdata 0x11e00. Vendor reads 66 bytes including
  the command, leaving 65 event bytes. The backend reads those 65 bytes,
  parses ten six-byte records, uses pressure high bytes at data offsets 62/63
  for the first two contacts, clamps pressure/width, swaps X/Y, reverses Y,
  reports BTN_TOUCH, and drops absent slots.

Intentional mainline differences:
- No vendor firmware loader, delayed update worker, /proc/NVTflash interface,
  display notifier, real-time kthread, gesture path, or vendor fallback
  dimensions are included. The firmware-update behavior observed in dmesg is
  therefore evidence only, not a permission to ship a writer.
- PID 0x0101 is retained as live evidence but is not used as a second match;
  the DT compatible and masked trim suffix select the disabled backend.
- The board node is not enabled. Touch rails, reset polarity, alternate
  address transition, firmware-info contents, and event frames still require
  a controlled hardware runtime probe.
