Observation boundary
--------------------

Established for the exact Candidate M/N kernel and DT foundation:

- the 11290000 T-PHY probe returned success;
- MTU3 at 11271000 bound with IRQ 208, high-speed limit, DMA32, and the logged
  FIFO geometry;
- the board's forced B-device session path executed;
- the built-in g_ether function registered with the pinned local MAC pair; and
- MTU3 logged its gadget pull-up action.

Not established:

- host detection, SET_ADDRESS, configuration selection, or USB enumeration;
- the electrical D+ state; the retained evidence is a driver log, not a line
  measurement or host observation;
- a host network interface, carrier, packet transfer, IP reachability, or a
  remote shell;
- which physical Gemini connector was used for these retained boot records;
- a real vusb33 supply (the probe explicitly used a dummy regulator);
- host mode, Type-C role switching, VBUS sourcing, or charging behavior.

Decision
--------

Do not rebuild the USB driver or DT merely to repeat probe registration. The
next USB serviceability candidate can retain an exact hardware-tested kernel
and DT, configure usb0 in its initramfs, and use a coordinated host observation
to distinguish these gates in order:

1. USB device appears with the pinned identity;
2. host network interface appears;
3. device and host report carrier;
4. fixed-address ICMP succeeds; and
5. the exact TCP marker and bounded remote-shell command/response succeed.

The initial shell may be unauthenticated only on the physically direct,
no-bridge laboratory link, with PID 1 retaining watchdog ownership and a finite
session/deadman; authentication is a later hardening delta. Failure must be
recorded at the first missing gate. Keep host/VBUS/Type-C and charging work
outside this gadget-only experiment.
