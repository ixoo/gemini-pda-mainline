The source-only analyzer passed against the current prepared tree. It
reconfirmed the vendor consys power/identity offsets, WMT/STP framing, BTIF
and legacy SDIO transport anchors, MT6797 Wi-Fi AP-DMA HIF, and Linux 7.1.3
BT/WMT/GNSS comparison sources. No code was copied from the vendor tree.

The current package does not enable a connectivity consumer. The reuse boundary
is standard STP/HCI, firmware-helper, cfg80211, serdev, SDIO, regulator, clock,
and GNSS framework code behind a new MT6797 consys owner and BTIF/legacy-SDIO
transport. Linux `btmtkuart` is a protocol reference only because the observed
active path is BTIF/SDIO, not a standard serdev UART. Linux `btmtksdio` is not a
drop-in match: Gemini evidence is 0x6628/0x6630/0x6632 function 2, a 4-byte
transport header, 512-byte blocks, and a 2080-byte FIFO versus Linux's newer
ID table, 5-byte header, and 256-byte blocks. `mt76` and generic `gnss-mtk`
are likewise not compatible by family name.
