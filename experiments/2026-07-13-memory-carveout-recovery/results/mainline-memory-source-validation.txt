The vendor MT6797 DTS has fixed ATF/preloader/LK/CCCI declarations and dynamic
CONSYS/SCP-share/SPM nodes; the generic Linux EVB DTS only declares a
contiguous memory range. The current Gemini patch preserves the fixed pre-LK
regions and all five dynamic reservation contracts. It does not snapshot the
post-LK mblock addresses observed in one live handoff.

The retained LK source audit additionally proves that LK rewrites `/memory`,
`/chosen`, model/CPU metadata, and Android firmware properties, then appends
runtime mblock reservations. Its `mblock_sanity_check()` compares existing
reserved-memory `reg` nodes against those runtime entries before appending them;
duplicating a post-LK snapshot in the input DT is therefore unsafe.

Decision: do not use the generic MT6797 EVB contiguous-memory declaration for
Gemini. Keep firmware-owned regions protected, preserve dynamic placement, and
resolve the final post-LK handoff before enabling modem, connectivity, SCP,
camera, display, or other DMA users.
