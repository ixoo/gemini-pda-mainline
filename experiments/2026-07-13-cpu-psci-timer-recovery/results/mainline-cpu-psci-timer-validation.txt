The generated DTB has ten CPU nodes, the generic PSCI 0.2 `smc` node, and
the four architectural timer PPIs. Unlike the vendor DT, it has no
`cpu-release-addr`, CPU idle-state nodes, per-CPU `clock-frequency`, explicit
PSCI function IDs, or timer `clock-frequency`. Those omissions match Linux
7.1.3's CPU/PSCI contracts and the timer binding's preference for
firmware-configured `CNTFRQ`; they are deliberate reuse decisions, not
missing vendor code. A focused pre-audit validation rejected the ten
per-CPU frequency properties because `cpus.yaml` sets `unevaluatedProperties:
false`; the local board patch now omits them.
